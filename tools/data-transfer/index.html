Data Transfer

curl

  • --limit-rate is optional and allows you to cap the data-transfer rate.
  • -L is optional and follows redirects.
  • -k is optional and skips SSL checks.
  • -o is optional and specifies an output file to store the response in, ex: output.txt.
  • https://example.com specifies the url you're requesting.

rSync

  • -r is optional and specifies that the rsync command should recurse into subdirectories.
  • -a is optional and specifies archive mode, meaning ACLs, extended attributes, and hard links should be preserved.
  • -P is optional and specifies that permissions should be preserved.
  • -e is optional and specifies a remote shell, in this example -- an SSH context for the rsync command.
  • src specifies where to read from.
  • target specifies where to copy to.

src and target may be remote filesystems, in which case they would take the format user@ip:/remote/path. A trailing slash will ensure the content of the directory will be copied from/to, rather than the directory itself.
